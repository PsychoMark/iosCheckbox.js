Usage

Include the css

<link rel="stylesheet" type="text/css" href="iosCheckbox.css" />

Add some checkboxes to your page

<input type="checkbox" class="ios" checked />
<input type="checkbox" class="ios" />

Load jQuery and iosCheckbox

<script type="text/javascript" src="http://code.jquery.com/jquery-1.11.3.min.js"></script>
<script type="text/javascript" src="iosCheckbox.min.js"></script>

Transform all checkboxs with the class of ios

<script type="text/javascript">
$(function()
{
    $('.ios').iosCheckbox();
});
</script>
Styles

The ‘slide’ style is the default:

It is possible to specify an alternative style. By default, the ‘check’ style is included:

There are two ways to specify a style for a checkbox. The first one is as an option in the iosCheckbox() call, to apply the style to all checkboxes in the ‘ios’ selector. 

<script type="text/javascript">
$(function()
{
    $('.ios').iosCheckbox({ style: 'check' });
});
</script>

Since the ‘ios’ class is used only as a selector and is not required for iosCheckbox to function, it is quite easy to set up two different styles by using a second class, for example ‘ios-check’. 

<script type="text/javascript">
$(function()
{
    $('.ios').iosCheckbox();
    $('.ios-check').iosCheckbox({ style: 'check' });
});
</script>

<input type="checkbox" class="ios-check" />

The second method is to specify the style on individual elements using a data-style attribute: 

<input type="checkbox" class="ios" data-style="check" />
Custom CSS classes

To customize the checkbox, like the color, you can provide a ‘class’ option to either the iosCheckbox() call, or through a data-class attribute: 

<input type="checkbox" class="ios" data-class="slide-customcolor" />
<input type="checkbox" class="ios" data-class="check-customcolor" data-style="check" />

In this example, parts of the ‘ios-ui-slide.checked’ and ‘ios-ui-check.checked’ styles from iosCheckbox.css have been copied to change the properties related to the color: 

.slide-customcolor.checked
{
  -webkit-box-shadow: inset 0 0 0 36px fuchsia;
  box-shadow: inset 0 0 0 36px fuchsia;
}

.check-customcolor.checked
{
  -webkit-box-shadow: inset 0 0 0 14px fuchsia;
  box-shadow: inset 0 0 0 14px fuchsia;
}

Using the ‘class’ option, the class is added to the iosCheckbox. To introduce a completely new style it is useful to know the class attribute of the iosCheckbox is set to 'ios-ui-[style]'.

Scripting

When calling iosCheckbox(), your original <input> element is hidden and new iosCheckbox elements takes its place. This is something to take into account when writing code or CSS that manipulates the checkbox.

iosCheckbox provides some compatibility out of the box:

  • The ‘checked’ state is synchronized between the original checkbox and the iosCheckbox.
  • The ‘change’ event is triggered on the original checkbox when clicking the iosCheckbox.

This also ensures <label> tags in forms work as intended:


<label for="exampleSlide">Form label example (click here to toggle) </label>
<input type="checkbox" class="ios" id="exampleSlide" />
